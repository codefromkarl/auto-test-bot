# Auto Test Bot — 系统宪章（System Charter）

## 使命
持续、自动、可解释地验证：真实用户是否仍然能够完成关键业务流程。

## 定位
Synthetic User Journey Monitoring 工程系统，而非测试平台或工作流平台。

## In Scope
- 真实浏览器执行（无 mock）
- Workflow / Phase / Action 建模
- 失败可解释
- MCP 仅作为证据采集观察者
- 面向决策的测试报告

## Out of Scope
- 通用工作流编排
- UI 自愈
- 内容质量评估
- API 单测 / 契约测试
- 测试管理 / SaaS 化

## 核心原则
Reality First / Failure-as-Signal / 可解释性高于成功率
