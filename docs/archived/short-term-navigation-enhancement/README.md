# 短期导航增强方案

## 概述

本文档描述了为了解决"加号按钮创建文生图"业务流程而实施的短期技术方案。该方案旨在最小化代码改动，快速解决当前问题，同时为未来的架构升级预留空间。

## 问题背景

当前 `navigate_to_text_image.py` 直接寻找"文生图"入口，不支持"先点击加号，再选择文生图"的业务流程。这种设计限制了机器人在实际业务场景中的适应性。

## 解决方案概述

通过配置增强和代码逻辑扩展，实现多策略导航支持：

1. **直接导航**：现有的直接文生图入口
2. **创建流程**：加号按钮 → 文生图菜单
3. **AI工具兜底**：现有的AI工具 → 文生图

- **风险等级**：低
